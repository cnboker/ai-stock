# Hermes Core Lessons - 长期核心记忆
更新日期: 2026-04-29（首次建立）

## 必须优先检查的系统性弱点

### 1. 幅度低估偏差（最高优先级）
- 模型普遍存在严重幅度低估问题：pred_change 通常仅为实际波动的 1/10 左右。
- 在强趋势行情中，容易大幅低估真实涨幅，导致信号保守失效。
- 重点检查：所有信号的 |Pred/Actual| 比例，以及 Pred/ATR 倍数（当前普遍 < 0.1x）。

### 2. Regime 分类器缺陷（第二优先级）
- Regime 分类器在连续趋势行情中容易错误输出 "neutral"，导致信号生成器切换到均值回归模式，甚至产生反向预测。
- 典型案例：价格已连续上涨数小时后，仍输出 neutral → 发出看空或过小看多信号。
- 必须检查：当市场出现明显单边趋势时，Regime 是否正确识别为 "good" 或趋势延续状态。

### 3. 不同波动率标的差异化处理
- **高波动/趋势标的**（如 sh603002）：
  - Gate 偏保守，建议在 model_score > 0.7 时允许更大的预测幅度。
- **低波动标的**（如 sh603871）：
  - score 在 0.54~0.60 区间容易产生低效微小交易（预测幅度极小，贡献负PnL）。
  - 建议对此类标的提升最低 score 硬阈值至 0.60。

### 4. 风控与信号层关系
- 风控层（equity_slope_break 等）有时能挽救信号层错误，但不能长期依赖风控掩盖信号层的方向性或幅度问题。
- 开盘竞价阶段（09:23 前后）信号需特别加强时间上下文过滤，避免将挂单堆积误判为趋势启动。

### 5. Gate 与 Score 逻辑
- 当前存在 Gate 悖论：final_score = raw_score * gate_mult 中，raw_score 是主要守门人。
- 拦截决策主要依赖 raw_score 而非 Gate_mult。

---

**审计原则（每次必须遵守）：**
- 每次复盘必须优先检查是否再次出现以上 1~3 条弱点。
- 明确指出「信号层 vs 风控层」的逻辑断层。
- 对高波动和低波动标的要区别诊断与建议。
- 给出具体、可执行的调优建议（而非泛泛而谈）。
