# Daily Audit Review Skill

## 目的
每天结束时，对当天的工作、项目进展、代码/系统/任务进行复查
## 执行步骤
1. 收集当天的工作日志、任务完成情况、问题记录等。
2. 进行复盘审计分析（成果、问题、教训、风险）。
3. **严格按照 templates/daily-audit-report.md 格式生成报告内容**。
4. **将报告写入独立文件**：
   - 文件路径：`reports/daily/YYYY-MM-DD-daily-audit-review.md`
   - 如果目录不存在，先创建目录。
5. 输出确认信息：报告已保存至 `[文件路径]`，并在对话中展示报告摘要。

## 输出要求
- 必须生成完整报告并保存为独立 Markdown 文件。
- 文件名格式：`YYYY-MM-DD-daily-audit-review.md`
- 同时在当前对话中输出报告的核心摘要（避免只保存不显示）。
## 何时使用
- 每天晚上或工作结束时
- 需要生成「每日复盘审计报告」时
- 回顾当天成果、问题、风险和行动计划

## 输入要求
- 当天主要工作内容、任务完成情况
- 遇到的主要问题、bug、风险
- 时间消耗记录（可选）
- 项目相关上下文（从 memory 或当前会话获取）

## 执行步骤
1. **信息收集**：回顾当天日志、任务、代码变更、会议记录等。
2. **审计分析**：
   - 成果与亮点
   - 问题与不足（技术、流程、效率）
   - 风险评估
   - 经验教训
3. **生成报告**：严格按照 `templates/daily-audit-report.md` 的格式输出。
4. **提取行动项**：列出明确、可执行的下一步行动（带负责人和截止时间）。
5. **保存报告**：将报告写入指定路径（如 reports/YYYY-MM-DD-audit.md）。

## 输出格式
必须使用 `templates/daily-audit-report.md` 中的模板。
报告应包含以下核心部分：
- 日期与概述
- 主要成果
- 审计发现（问题分类）
- 改进建议
- 行动计划（Action Items）
- 下次重点关注事项

## 最佳实践
- 保持客观、具体、可量化
- 区分事实与分析
- 优先级高的风险用 **高/中/低** 标注
- 行动项要 SMART（具体、可衡量、可达成、相关、有时限）
- 每周/每月可对历史报告做总结

## 相关模板
